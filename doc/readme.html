1) Descripción general

IPN_TFTn es un proceso de transistor de película delgada tipo n, implementado en arquitectura bottom-gate / top-contact, orientado a la fabricación y evaluación de TFTs basados en semiconductor IGZO.

El stack principal se compone de:

  • Substrato (dominio mecánico/geométrico de diseño),
  • Cr/Au como compuerta inferior,
  • Al₂O₃ como dieléctrico de compuerta,
  • IGZO como capa semiconductora activa,
  • Mo para source/drain e interconexión superior,
  • PMMA como pasivación con aperturas.

El PDK actual incluye tecnología KLayout, archivo de capas, deck DRC, deck de extracción LVS/SPICE, visualización 2.5D, PCell para transistor TFT y PCell de vías/contactos.

Aplicaciones objetivo: estructuras TFT de prueba, celdas básicas, caracterización eléctrica y simulación de circuitos a partir de un modelo equivalente tipo MOS utilizado como aproximación compacta del dispositivo TFT.

3.2 Relación capa ↔ máscara de fabricación

  • Máscara G (gate): define 2/0 Cr/Au.drawing.
  • Máscara D (dieléctrico/contacto): define 3/0 Oxide.cut.
  • Máscara S (semiconductor): define 4/0 IGZO.drawing.
  • Máscara M1 (metal superior): define 5/0 Mo.drawing.
  • Máscara P (pasivación): define 6/0 PMMA.cut.
  • Capas auxiliares CAD/PDK: 1/1, 4/1, 5/1 y 5/2.
La capa 4/1 tft.marker no representa una capa física del proceso: se utiliza como marcador geométrico interno para la PCell y la extracción LVS del TFT.

4) Derivación geométrica interna (DRC/LVS). Solo para uso interno LNunT

El extractor y los scripts auxiliares definen las siguientes capas derivadas:

  • cont_sem = met1 & semi → intersección metal/semiconductor, usada para source/drain.
  • cont_gat = gate & semi → intersección gate/semiconductor, usada como referencia geométrica del control de compuerta.
  • res_term = met1 & mpin → terminales definidos por puerto.
  • res_body = met1 - res_term → porción conductiva restante del metal.

Interpretación La conectividad actual del extractor usa la superposición geométrica como criterio principal y agrega una referencia global implícita a GND para la capa semiconductora, consistente con el uso actual de un modelo MOS equivalente para aproximar el TFT.

5) Reglas de diseño (DRC) vigentes

5.1 Contención en dominio de substrato

  • mo_metal.outside(substrate)
  • igzo.outside(substrate)

5.2 Reglas mínimas sobre metal de Mo

  • Ancho mínimo: 5 µm
  • Espacio mínimo: 5 µm
  • Muesca mínima: 5 µm

5.3 Reglas de borde

  • substrate.enclosing(mo_metal,50.um) → separación mínima a borde del área de diseño: 50 µm.

5.4 Integridad geométrica

  • mo_metal.odd_polygons
  • mo_port.odd_polygons

5.5 Reglas sobre etiquetado

  • mo_labels.width(5000.um) → detección de geometrías dibujadas indebidamente en la capa de etiquetas.
El deck DRC actual es funcional para un primer control geométrico del proceso, especialmente sobre la capa metálica superior y la pertenencia al dominio de diseño.

6) Extracción LVS y netlist

6.1 Dispositivo extraído

  1. TFT tipo n: se extrae como IPN_TFTN, utilizando una representación compacta basada en geometría.

6.2 Mapeo de terminales en extracción

  • "SD" => cont_sem → región de contacto source/drain en superposición Mo–IGZO.
  • "G" => tftg → terminal de compuerta gobernado por el marcador 4/1.
  • "W" => semi → cuerpo geométrico del semiconductor.
  • "tD", "tS" y "tG" → terminales auxiliares geométricas para el extractor.
  • "tB" => semi → cuerpo/bulk equivalente, asociado a semiconductor.

6.3 Conectividad y exportación

  • Conexión de etiquetas metálicas: connect(met1, mlbl).
  • Conexión de puertos: connect(res_term, mlbl).
  • Referencia global: connect_global(semi,"GND") y connect_implicit("GND").
  • Purga de netlist: netlist.purge y netlist.purge_nets.
  • Salida SPICE: archivo .cir con nombre derivado automáticamente de la celda top.
  • Generación de librería de modelos: macro separado que invoca WriteModels.create_models(output_dir).

6.4 Modelo SPICE suministrado

La librería del proceso incorpora actualmente la siguiente definición compacta para simulación:

Modelo activo: .model IPN_TFTN NMOS ( LEVEL=1 VTO=1.08518 KP=7.58022e-08 LAMBDA=0.121604 GAMMA=0 PHI=0.7 RD=0 RS=0 CGSO=0 CGDO=0 )

Al usar este modelo con el extractor mos4("IPN_TFTN"), el cuarto terminal se conserva en el netlist, pero en el flujo actual se asume una referencia global implícita a GND para el cuerpo equivalente asociado al semiconductor.

7) Flujo recomendado de diseño (usuario)

  1. Definir el dominio de diseño en 1/0 Sub_Si-p.drawing.
  2. Dibujar la compuerta inferior en 2/0 Cr/Au.drawing.
  3. Definir cortes/contactos dieléctricos en 3/0 Oxide.cut cuando aplique.
  4. Dibujar la región semiconductora en 4/0 IGZO.drawing.
  5. Colocar el marcador del transistor en 4/1 tft.marker para definición de canal y extracción.
  6. Dibujar source/drain e interconexiones en 5/0 Mo.drawing.
  7. Definir puertos en 5/2 Mo.port y etiquetas de red en 5/1 Mo.text.
  8. Definir aperturas de pasivación en 6/0 PMMA.cut.
  9. Ejecutar DRC y corregir violaciones.
  10. Ejecutar LVS/extracción y revisar netlist SPICE.
  11. Generar o copiar la librería de modelos y simular el circuito.

8) Restricciones y buenas prácticas

  • No dejar geometrías de IGZO ni Mo fuera del dominio substrate.
  • Respetar 5 µm en ancho/espacio/muesca de la capa Mo.drawing según el deck actual.
  • Mantener una separación mínima de 50 µm al borde del substrato para el metal superior.
  • Usar tft.marker de forma consistente para que la extracción reconozca correctamente el dispositivo.
  • Definir puertos mediante Mo.port y nombres de red mediante Mo.text.
  • Evitar polígonos con autotraslape en metal y puertos.
  • Documentar claramente cuando el bulk equivalente del modelo se asuma a GND en simulación.

10) Observaciones técnicas previas a release

Punto 1: La vista 2.5D actual no coincide con los espesores físicos nominales. La escala se utiliza con el objetivo de mejorar la visualización de dispositivos de capa delgada.
Punto 2: La capa 3/0 Oxide.cut aparece nombrada como corte de óxido, pero en la vista 2.5D también se usa como contact. En este caso se utilizan como sinónimos.
Punto 3: La extracción LVS usa mos4("IPN_TFTN") y referencia implícita a GND para el semiconductor/bulk equivalente. Con el modelo SPICE actual LEVEL=1, esta aproximación es útil para circuitos y validación inicial, pero sigue siendo un modelo sustituto del TFT y no un modelo físico nativo del material IGZO.
Punto 4: El PDK ya contiene una PCell funcional para el transistor y una PCell de vías con al menos una definición explícita de vía V1 entre Cr/Au (2/0) y Mo (5/0) usando corte 3/0. Sería conveniente integrar sus reglas mínimas en el deck DRC oficial.
Punto 5: La capa PMMA.cut (6/0) participa en la vista de proceso, pero actualmente no tiene reglas DRC explícitas en el deck entregado. Puede añadirse verificación de contención, separación y cobertura mínima para tapeout más robusto.
